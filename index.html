JAMstack Background

To understand JAMstack's role today, we need to:

  • Take a look at history of web development
  • How we got here

Web Development in the Early 1990s

Web Development was simple

  • Web pages initially created as document management system.
  • They were hand coded in HTML containing links to other pages.
  • Browsers initially did not have multimedia support.

Web Development in Mid 1990s to Early 2000s

Web Development Evolving

  • Web pages are not hand-coded but are generated dynamically by programming languages.
  • They require application server/language runtime.
  • With dynamic sites, servers would get content from databases and render web pages in real-time for each user, instead of building html pages ahead of time.
  • May Require databases

Early Examples of Web Development From This Era

  • Amazon in 1994
  • Yahoo in 1994
Complexity of Dynamic Sites Brings About Popularity of Content Management Systems and Proliferation of Blogging Software
  • LiveJournal: 1999
  • Blogger: 1999
  • MoveableType: 2001
  • Drupal: 2001
  • MediaWiki: 2001
  • WordPress: 2003
  • Joomla: 2005
  • Magento: 2009
Web Dev Evolution Also Impacts Client Side and Dynamic Web Page Creation
  • JavaScript in 1995
  • DHTML in 1997: Use of JS to change the look of the page - showing/hiding elements, animations.
  • AJAX in 2006: Use of JS to retrieve and send data in the background without reloading the page.

Dynamic Sites Have These

Pros Cons
Content is updated in real time. More Software, more complexity, more maintenance
Content can be customized for each user

Web Development in Mid 2000s to Present

Web Development Changing, Again:

Developers Wanted Easier Process

Static Site Generators, SSG

  • A program/script that would take content stored in a format like XML and apply a template to that content to build static html files.
  • Has existed since the 1990s: MovableType had an option where you can build a static type.
  • Became more popular in the 2000s due to the invention of lightweight markup languages.
    • reStructuredText in 2002
    • Textile in 2002
    • markDown in 2002

Examples of SSGs

  • Jenkyll in 2008
  • Pelican in 2010
  • Hugo in 2013

Static Sites Generators Have These

Pros Cons
Simple; less software, less complexity Limitation on the number of authors
Limitation on the number of content
Developers Demand for Easier Process Led to Rise of Single Page Applications (SPA)
  • Most of HTML is rendered in browser, not in server
  • Data can be fetched asynchronously via API calls
  • Decoupled from back end
  • Flat file web sofware
  • Examples of JS frameworks used for SPAs
    • Angular, 2010
    • EmberJS, 2011
    • React, 2013
    • VueJS, 2014

Single Page Apps Have

Pros Cons
Decoupled; separation of concerns. Asking for more from the browser; problems for low-end phones, battery-life on mobile
Easy to host; only need a web server

JAMstack: JavaScript + APIs + Markup

APIs are used to connect and pull external data, making static sites to become more dynamic.
