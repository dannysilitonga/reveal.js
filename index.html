Web Development in Early 1990s

In 1990s, Web Development Was Simple

  • Web pages initially created as document management system
  • They were hand coded in HTML containing links to other pages
  • Browsers initially did not have multimedia support

Web Development in Mid 1990s to Early 2000s

Web Development Changing

  • Web pages were not hand-coded but were generated dynamically by programming languages
  • They would require application server/language runtime, producing dynamic web pages
  • With dynamic sites, servers would get content from databases and render web pages in real-time for each user, instead of building html pages ahead of time
  • May Require databases

Examples of Dynamic Sites From This Era

  • Amazon in 1994
  • Yahoo in 1994
Also in This Era, Complexity of Developing Dynamic Sites Brings About Popularity of CMS and Proliferation of Blogging Software

Examples of Content Management Systems, CMS:

  • LiveJournal in 1999
  • Blogger in 1999
  • MoveableType in 2001
  • Drupal in 2001
  • MediaWiki in 2001
  • WordPress in 2003
  • Joomla in 2005
  • Magento in 2009

Changes Introduced New Tools on Client Side

  • JavaScript in 1995
  • DHTML in 1997: Use of JS to change the look of the page - showing/hiding elements, animations
  • AJAX in 2006: Use of JS to retrieve and send data in the background without reloading the page

Advantages and Disadvantages of Dynamic Sites

Pros Cons
Content is updated in real time More Software, more complexity, more maintenance
Content can be customized for each user

Web Development in Mid 2000s to Present

Web Development Changing, Again

Developers wanted easier time to develop and deploy

This led to rise of Modern Static Site Generators, SSG

  • Scripts that would take content stored in format like XML and apply template to that content to build static html files
  • Has existed since the 1990s (MovableType)
  • Popular in 2000s due to lightweight markup languages: reStructuredText in 02, Textile in 02, markDown in 04
  • Examples: Jenkyll in 08, Pelican in 10, Hugo in 13

Advantages and Disadvatages of SSGs

Pros Cons
Simple; less software, less complexity Limitation on the number of authors
Limitation on the number of content
Not Just SSGs, Single Page Apps Are Also Popular
  • Most of HTML is rendered in browser, not in server
  • Data can be fetched asynchronously via API calls
  • Decoupled from back-end
  • Flat file web sofware
  • Examples of JS frameworks used for SPAs: Angular, 2010; EmberJS, 2011; React, 2013; VueJS, 2014

Advantages and Disadvatages of Single Page Apps

Pros Cons
Decoupled; separation of concerns Asking for more from the browser; problems for low-end phones, battery-life on mobile
Easy to host; only need a web server
In Mid 2010s, as the Tech Matures, We Have JAMstack

JavaScript + APIs + Markup

APIs are used to connect and pull external data, making static sites to become more dynamic
